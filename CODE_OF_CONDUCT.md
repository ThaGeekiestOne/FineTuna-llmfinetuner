# Code of Conduct

## Our Pledge

FineTuna is an open-source project built to make LLM fine-tuning more accessible. We are committed to providing a welcoming, respectful, and harassment-free environment for everyone, regardless of experience level, background, identity, or perspective.

## Expected Behavior

Project participants are expected to:

- Use welcoming and inclusive language.
- Respect differing viewpoints and technical opinions.
- Give and receive constructive feedback professionally.
- Focus criticism on ideas, code, documentation, and project outcomes, not people.
- Be patient with newcomers and people learning the project.
- Respect maintainers' time and decisions.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language, imagery, or unwanted attention.
- Trolling, insulting comments, or sustained disruption.
- Publishing private information without explicit permission.
- Repeatedly ignoring maintainer requests or project boundaries.

## Scope

This Code of Conduct applies in all project spaces, including GitHub issues, pull requests, discussions, documentation, social channels, community chats, and any public or private communication related to FineTuna.

It also applies when someone is representing the project publicly, such as through an official account, event, demo, or published project material.

## Enforcement

Maintainers are responsible for clarifying and enforcing this Code of Conduct. Depending on the situation, maintainers may take actions including:

- Asking someone to correct or stop harmful behavior.
- Editing, hiding, or deleting comments, issues, discussions, or pull requests.
- Temporarily restricting participation.
- Permanently banning someone from project spaces.
- Reporting serious abuse to the relevant platform or authority.

Enforcement decisions will be made based on the impact of the behavior, the safety of the community, and the needs of the project.

## Maintainer Responsibilities

Maintainers should apply this Code of Conduct consistently and fairly. Maintainers who violate or ignore this Code of Conduct may be removed from project responsibilities.

## Attribution

This Code of Conduct is adapted from common open-source community standards, including the Contributor Covenant.
